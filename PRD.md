# PRD: 蝴蝶人生 - 开放式人生偏航叙事小程序

## 1. 产品定位

“蝴蝶人生”是一个类海龟汤气质的微信原生小程序叙事产品。

用户先得到或创建一个普通人，再让这个普通人在生活半径中遇到一件看似很小的事。之后，用户每一步都从若干选择中选一个行动，系统根据角色的人格惯性、选择后果、现实压力和偏航状态，实时续写下一段人生。

它不是传统分支剧情，也不是预设结局游戏。它的核心体验是：

> 一个普通人因为一次微小选择，开始偏离原本的人生惯性。她可能回到原处，也可能慢慢走向完全不可思议的命运。

## 2. 产品初衷

产品希望让用户：

1. 看到不一样的人生。
2. 看到不同选择带来的不同人生。
3. 感受到一个人并不会因为一次勇敢立刻改变，改变需要反复选择和承担代价。
4. 在别人的人生里，间接看见自己的惯性、恐惧、欲望和可能性。

## 3. 核心体验

### 3.1 一句话体验

先给我一个人，再给她一天里发生的一件小事。剩下的，让她自己活下去。

### 3.2 用户每次进入一局故事时

用户需要完成两个起点动作：

1. 创建或选择一个初始角色。
2. 创建或随机获得一个起点事件。

然后进入连续叙事循环：

1. 系统讲述上一选择后的剧情。
2. 系统展示角色当下的内心反应与现实压力。
3. 系统给出几个下一步选择。
4. 用户选择一个行动，或自行输入行动，或选择“到此为止”。
5. 如果用户没有选择“到此为止”，故事继续。

## 4. MVP 范围

MVP 要验证的不是“故事能不能无限宏大”，而是：

> 用户是否愿意陪一个普通人，在一次次小选择里，看到她逐渐偏离原本人生。

### 4.1 MVP 必须包含

1. 角色创建入口
   - 用户自行填写人物画像。
   - 系统随机给出若干初始角色卡，用户选择其一。

2. 起点事件入口
   - 用户自行填写事件。
   - 系统根据初始角色随机生成一个生活半径内的小事件。

3. 角色底盘
   - 当前身份。
   - 人格惯性。
   - 深层欲望。
   - 最大恐惧。
   - 道德底线。
   - 关系牵引。
   - 资源状态。
   - 六个自我。

4. 连续叙事回合
   - 每回合一小段剧情。
   - 每回合若干选择。
   - 支持用户自定义行动。
   - 支持“到此为止”并生成阶段性回望。

5. 隐藏状态盘
   - 惯性。
   - 瞬时偏航。
   - 人格沉积。
   - 现实压力。
   - 世界异化度。

6. 阶段性回望
   - 当用户选择结束时，系统总结这段人生走向。
   - 回望不做胜负评价，只描述她变成了怎样的人，失去了什么，获得了什么。

### 4.2 MVP 不包含

1. 大量预设剧情树。
2. 装备、地图、战斗、数值养成等传统游戏系统。
3. 多角色并行人生模拟。
4. 复杂世界观设定。
5. 开局直接进入总统、间谍、杀手、穿越等高离谱状态。
6. 技术实现、接口设计、数据库设计、模型选型。

## 5. 核心概念

### 5.1 人格底盘

人格底盘用于保证角色始终像同一个人，而不是被 AI 随机换皮。

每个角色至少包含：

1. 当前身份  
   例如：朝九晚八的打工人、全职妈妈、失业青年、县城公务员、独居老人。

2. 人格惯性  
   角色在压力下最常见的反应方式。  
   例如：讨好、逃避、沉默、忍耐、合理化、自责、冲动、控制。

3. 深层欲望  
   角色真正想要但不一定敢承认的东西。  
   例如：自由、被看见、被爱、证明自己、掌控生活、摆脱羞耻。

4. 最大恐惧  
   角色最怕失去或面对的东西。  
   例如：失业、被讨厌、惹麻烦、孤独、失控、贫穷、被抛弃。

5. 道德底线  
   角色当前绝不会轻易做的事。  
   这个底线可以随着故事变化，但必须有代价和过程。

6. 关系牵引  
   谁会拉住她，谁会推动她，谁会诱惑她，谁会惩罚她。

7. 资源状态  
   钱、时间、健康、技能、人脉、名声、信息。

8. 六个自我  
   以威廉·詹姆斯的自我理论作为上层人物理解框架：
   - 主体我：正在感受、判断、行动的她。
   - 客体我：她眼中“我是怎样的人”。
   - 物质自我：身体、衣物、房间、财产、物件、生活质感。
   - 社会自我：她在他人眼中的身份和评价。
   - 精神自我：她的价值、欲望、信念、羞耻、尊严。
   - 纯粹自我：贯穿变化的“我还是我”的连续感。

### 5.2 起点事件

起点事件必须是角色生活半径中合理发生的小事。

事件公式：

> 日常场景 + 轻微异常 + 道德/欲望/恐惧压力

例子：

1. 她过马路时扶起一个摔倒的老人，但她马上要迟到了。
2. 她在公司打印机里看到一份裁员名单，上面有唯一帮过她的同事。
3. 她收到一笔陌生转账，备注是：“谢谢你刚才没说出来。”
4. 主管让她背一个不是她造成的锅，只要她点头，这个月绩效就保住。
5. 地铁上，一个陌生人把她错认成另一个人，并低声说：“你终于来了。”

起点事件不需要宏大，但必须能撬动角色的自我结构。

### 5.3 偏航度

偏航度描述角色和世界偏离原本日常轨道的程度。

偏航度不是单向升级条，而是会回弹的。

它至少拆成两层：

1. 瞬时偏航  
   某一刻角色做出了不符合惯性的选择。  
   例如：胆小的人第一次反驳主管。

2. 人格沉积  
   这次选择是否真的留下痕迹。  
   例如：她不只是反驳了，还在付出代价后没有否定自己。

偏航规则：

1. 反惯性选择会提高瞬时偏航。
2. 承担代价后仍保持自我认可，会提高人格沉积。
3. 遇到惩罚后退回旧模式，偏航会回落。
4. 重复多次的新选择，会形成新的惯性。
5. 人格沉积足够多时，角色才真正改变。

### 5.4 世界异化度

世界异化度描述故事允许多么离谱。

初始世界应接近现实。随着偏航和人格沉积增加，世界可以逐步允许更高类型化的事件进入：

1. 现实生活。
2. 悬疑巧合。
3. 黑色幽默。
4. 爱情命运感。
5. 都市奇谈。
6. 科幻异常。
7. 穿越、古装、平行世界。
8. 权力、间谍、犯罪、英雄化人生。

世界异化度高，只代表“门可以出现”，不代表角色一定会走进去。

## 6. 叙事循环

### 6.1 每回合标准结构

每一回合应包含：

1. 上一步选择后的外部剧情。
2. 角色的内心反应。
3. 现实给她的回拽或诱惑。
4. 3 个系统选项。
5. 1 个自定义行动入口。
6. 1 个“到此为止”入口。

### 6.2 选项设计原则

选项不应是简单的正确/错误，而应代表不同自我的争夺。

常见选项类型：

1. 保持安全。
2. 说出真话。
3. 逃避。
4. 冒险。
5. 讨好别人。
6. 保护弱者。
7. 追逐欲望。
8. 越界。
9. 求助。
10. 隐瞒。

同一个选择可能既保护她，也伤害她。系统不应用明显的道德打分提示用户。

### 6.3 每次选择后的三类结果

每次选择后，系统必须更新三类结果：

1. 外部结果  
   现实世界发生了什么。

2. 内部结果  
   角色如何理解这次选择，是否改变了自我叙事。

3. 世界结果  
   他人、环境、关系、机会和风险如何回应她。

## 7. 走向控制规则

### 7.1 不完全依赖 AI

产品不应把故事走向完全交给 AI 自由发挥。

AI 负责即兴续写，系统负责命运的物理法则。

也就是说：

1. AI 负责让下一场戏鲜活、具体、有想象力。
2. 规则负责判断它是否符合角色、是否欠因果、是否有代价、是否飞得太快。

### 7.2 离谱事件必须偿还因果债

越离谱的走向，越需要因果债。

例如：

1. 成为总统  
   需要公共表达、组织能力、群众投射、权力欲、平台、时代机会。

2. 成为 FBI 间谍  
   需要秘密、双重身份、被机构注意、可利用价值、能承受谎言。

3. 成为赌徒  
   需要逃避现实、风险快感、一次侥幸收益、债务或欲望循环。

4. 成为连环杀人犯  
   需要严重道德坍塌、去人化、失控升级、不可逆越界、掩盖机制，并且必须写出恐怖和代价，不能美化。

5. 成为昼伏夜出的城市守护者  
   需要保护欲、技能积累、创伤或誓言、双重生活、牺牲成本、持续风险。

### 7.3 类型门槛

不同类型走向必须满足门槛。

1. 悬疑门槛  
   出现未解释的信息、重复巧合、隐瞒者。

2. 爱情门槛  
   稳定互动、误解或吸引、脆弱暴露。

3. 科幻门槛  
   异常证据、规则重复、技术或时间裂缝。

4. 权力门槛  
   公共冲突、组织资源、被人追随、话语权上升。

5. 犯罪门槛  
   越界收益、道德钝化、风险耐受、掩盖能力。

6. 英雄门槛  
   保护欲、技能积累、牺牲成本、双重生活。

门槛未满足时，故事不能硬拐。

### 7.4 现实回拽

每 3-5 步应出现一次现实回拽。

现实回拽可以来自：

1. 钱。
2. 工作。
3. 身体疲惫。
4. 家庭电话。
5. 亲密关系。
6. 法律风险。
7. 名声。
8. 内疚。
9. 睡眠不足。
10. 旧习惯。

现实回拽的作用是防止故事变成无重力爽文。

### 7.5 人格不变量

无论故事走到多离谱，角色必须保留某个深层连续性。

例如：

1. 她永远害怕被抛下。
2. 她永远想证明自己不是废物。
3. 她永远对弱者心软。
4. 她永远不敢直视自己的愤怒。

人格不变量让总统、赌徒、间谍、黑暗人物都仍然是“她”。

## 8. 阶段性回望

故事不设置传统结局。

只要用户不选择“到此为止”，故事可以继续。

用户选择“到此为止”时，系统生成阶段性回望，内容包括：

1. 她现在更像谁了。
2. 她失去了什么。
3. 她获得了什么能力、关系或阴影。
4. 哪个自我变强了。
5. 哪个旧自我还在暗处拖她。
6. 这段人生最重要的一次选择是什么。

回望不应评价用户“玩得好不好”，而应像一段人生切片。

## 9. 示例 MVP 主线

### 9.1 初始角色

她是一个唯唯诺诺的打工人，每天朝九晚八。

她厌倦这样的生活，但没有勇气突破。

她的人格惯性是讨好、忍耐、自责和回避冲突。

她的深层欲望是自由和被尊重。

她的最大恐惧是失业、惹麻烦、被别人讨厌。

### 9.2 起点事件

她上班路上，在过马路时扶起一个摔倒的老奶奶。

这不是一个被社会完全鼓励的选择，因为她担心被讹，也担心迟到。

### 9.3 可能的发展方式

她可能因为迟到被主管责备。

她可能第一次说：“我刚才扶了一个摔倒的人，我不觉得这件事需要道歉。”

她也可能话到嘴边又咽回去，回到旧惯性。

如果她持续承担代价，并把微小勇敢沉积下来，多年后她可能走向非常离谱但仍有因果的方向：

1. 成为维权者。
2. 成为城市暗处的调查者。
3. 被卷入一个悬疑组织。
4. 发展出双重身份。
5. 成为昼伏夜出的城市守护者。

但她也可能回落：

1. 继续胆小如鼠。
2. 变得更犬儒。
3. 成为投机者。
4. 因恐惧与欲望走向赌徒或犯罪。

关键不是系统预设她会成为什么，而是每一步选择是否改变了她。

## 10. 设计边界

### 10.1 设计应强调

1. 读故事和做选择的沉浸感。
2. “小事正在改变人生”的微妙感。
3. 角色不是用户傀儡，而是有惯性的人。
4. 隐藏状态可以被用户感受到，但不需要全部数值化展示。
5. 结束时的回望要有文学性和人生感。

### 10.2 设计应避免

1. 游戏化过重导致像数值养成。
2. 选项过于道德题或心理测试题。
3. 一开始就给用户太多设定负担。
4. 让离谱类型元素来得太快。
5. 让用户感觉 AI 在随机胡编。

## 11. 开发边界

当前 PRD 不讨论技术实现。

开发 agent 后续需要根据本 PRD 自行拆解：

1. 数据结构。
2. 状态保存。
3. AI 调用策略。
4. 提示词架构。
5. 安全策略。
6. 小程序页面结构。
7. 内容审核与风险控制。

但开发时必须保留本 PRD 的上层叙事原则：

1. 角色有惯性。
2. 偏航会回弹。
3. 离谱要偿还因果债。
4. 每 3-5 步有现实回拽。
5. 故事可以无限继续。
6. 用户结束时才生成阶段性回望。

## 12. 成功标准

MVP 成功不以故事数量或世界观宏大程度衡量。

更重要的成功标准是：

1. 用户玩 10 步以后，仍觉得这是同一个人。
2. 用户感觉自己影响了她，但没有完全操控她。
3. 角色有时勇敢，有时回缩。
4. 每个选择都留下余震。
5. 离谱事件出现时，用户觉得意外但合理。
6. 用户点“到此为止”时，能感到自己看完了一段人生岔路。

## 13. 给后续 Agent 的分工建议

### 13.1 设计 Agent

请基于本 PRD 重点设计：

1. 开局角色创建体验。
2. 随机角色卡的呈现方式。
3. 起点事件选择体验。
4. 单回合叙事阅读与选择界面。
5. 自定义行动入口。
6. “到此为止”的阶段性回望界面。
7. 隐藏状态如何以克制方式被用户感知。

设计不需要展开技术实现。

### 13.2 开发 Agent

请基于本 PRD 重点设计：

1. 如何表示角色底盘。
2. 如何保存连续故事状态。
3. 如何控制偏航度、人格沉积、现实压力和世界异化度。
4. 如何约束 AI 不乱飞。
5. 如何生成每回合选项。
6. 如何处理用户自定义行动。
7. 如何生成阶段性回望。

开发不应把本产品做成固定剧情树，而应做成受规则约束的开放叙事系统。
